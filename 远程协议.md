# 嘉慧功率计控制程序 — 远程协议说明

本文档整理当前软件涉及的两层通信协议：

1. **TCP 自动化接口**：供外部程序通过网络远程控制本软件（连接设备、读功率、记录数据等）。
2. **VISA/SCPI 设备协议**：本软件与 Thorlabs PM100D 功率计硬件之间的底层指令。

---

## 一、TCP 自动化接口

### 1.1 概述

| 项目 | 说明 |
|------|------|
| 协议类型 | TCP |
| 默认端口 | `1235`（在 `PM100D_Control.py` 中硬编码） |
| 字符编码 | UTF-8 |
| 消息分帧 | 每条消息以换行符 `\n` 结尾 |
| 数据格式 | JSON |
| 连接模式 | 客户端主动连接，服务器被动响应（一问一答） |
| 并发 | 支持多客户端同时连接，每个连接独立线程处理 |
| 启动时机 | 软件启动时自动开启，退出时自动关闭 |

> 实现文件：`TCPServer.py`（服务端）、`TCPClient.py`（客户端参考实现）、`PM100D_Control.auto_server_controller`（命令分发）。

### 1.2 通信流程

```
客户端                          服务端（本软件）
  |                                  |
  |-------- TCP 连接 -------------->|
  |                                  |
  |-- JSON 请求 + \n --------------->|
  |                                  | 解析 opcode，执行业务逻辑
  |<-- JSON 响应 + \n ---------------|
  |                                  |
```

- 客户端发送一条完整 JSON 字符串，末尾加 `\n`。
- 服务端处理完毕后，返回一条 JSON 响应字符串，末尾同样加 `\n`。
- 若 JSON 解析失败或 `opcode` 未知，返回 `isSuccess: false` 及错误信息。

### 1.3 通用响应格式

所有接口均返回如下结构的 JSON：

```json
{
  "isSuccess": true,
  "Value": "<响应内容>",
  "ErrorMessage": "<错误信息>"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `isSuccess` | bool | 操作是否成功 |
| `Value` | string / object | 成功时的返回值；无数据时可为空字符串 `""` |
| `ErrorMessage` | string | 失败时的错误描述；成功且无错误时通常为 `"Null"` 或 `""` |

### 1.4 命令列表

#### 1.4.1 获取功率 — `GetPower`

读取界面当前显示的 CH1 功率值（LCD 显示值，已按界面规则格式化后的字符串）。

**请求：**

```json
{
  "opcode": "GetPower"
}
```

**成功响应示例：**

```json
{
  "isSuccess": true,
  "Value": {
    "CH1": "1.23456"
  },
  "ErrorMessage": "Null"
}
```

| 字段 | 说明 |
|------|------|
| `Value.CH1` | 通道 1 当前功率显示值（字符串） |

---

#### 1.4.2 记录控制 — `RecordCon`

控制数据记录的开始与停止。记录文件保存在程序目录下 `./Record/` 文件夹，CSV 格式，编码 GBK。

**开始记录：**

```json
{
  "opcode": "RecordCon",
  "parameter": {
    "Con": "Start"
  }
}
```

**停止记录：**

```json
{
  "opcode": "RecordCon",
  "parameter": {
    "Con": "Stop"
  }
}
```

| 参数 | 可选值 | 说明 |
|------|--------|------|
| `parameter.Con` | `"Start"` | 开始记录（若已在记录中则返回失败） |
| `parameter.Con` | `"Stop"` | 停止记录并重命名 CSV 文件（若未在记录则返回失败） |

**成功响应：**

```json
{
  "isSuccess": true,
  "Value": "",
  "ErrorMessage": ""
}
```

**失败响应示例：**

```json
{
  "isSuccess": false,
  "Value": "",
  "ErrorMessage": "开启记录错误!"
}
```

```json
{
  "isSuccess": false,
  "Value": "",
  "ErrorMessage": "关闭记录错误!"
}
```

**不支持的 Con 值：**

```json
{
  "isSuccess": false,
  "Value": "",
  "ErrorMessage": "command not supported:{...}"
}
```

#### 1.4.3 连接设备 — `ConnectDevice`

自动扫描可用 VISA 设备并连接界面下拉框中当前选中的设备。

> 注意：连接的是 GUI 中 `Com` 下拉框当前选中的设备地址，请确保在调用前已在界面选好目标设备，或依赖默认选中项。

**请求：**

```json
{
  "opcode": "ConnectDevice"
}
```

**成功响应：**

```json
{
  "isSuccess": true,
  "Value": "",
  "ErrorMessage": ""
}
```

**失败响应：**

```json
{
  "isSuccess": false,
  "Value": "",
  "ErrorMessage": "连接设备错误!"
}
```

执行逻辑：先调用 `check_port_callback()` 刷新设备列表，再执行 `connect_pm()`。

---

#### 1.4.4 版本查询 — `check`

查询当前软件版本号（从 `更新内容.csv` 最后一行读取）。

**请求：**

```json
{
  "opcode": "check"
}
```

**成功响应示例：**

```json
{
  "isSuccess": true,
  "Value": "v1.0.3",
  "ErrorMessage": "Null"
}
```

---

#### 1.4.5 未知命令

当 `opcode` 不在上述列表中时：

```json
{
  "isSuccess": false,
  "Value": "",
  "ErrorMessage": "Unknown command!"
}
```

### 1.5 客户端示例（Python）

```python
import json
import socket

HOST = "127.0.0.1"   # 运行功率计软件的机器 IP
PORT = 1235

def send_command(opcode, parameter=None):
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.connect((HOST, PORT))
    req = {"opcode": opcode}
    if parameter is not None:
        req["parameter"] = parameter
    sock.sendall((json.dumps(req) + "\n").encode("utf-8"))
    data = b""
    while b"\n" not in data:
        data += sock.recv(1024)
    sock.close()
    return json.loads(data.decode("utf-8").strip())

# 查询版本
print(send_command("check"))

# 连接设备
print(send_command("ConnectDevice"))

# 读取功率
print(send_command("GetPower"))

# 开始/停止记录
print(send_command("RecordCon", {"Con": "Start"}))
print(send_command("RecordCon", {"Con": "Stop"}))
```

## 二、VISA/SCPI 设备协议

### 2.1 概述

本软件通过 **PyVISA** 经 **USB** 与 Thorlabs PM100D 系列功率计通信，指令遵循 **SCPI**（Standard Commands for Programmable Instruments）标准。

| 项目 | 说明 |
|------|------|
| 传输层 | USB VISA |
| 指令标准 | SCPI |
| 后端库 | pyvisa / pyvisa-py |
| 超时 | 5000 ms（连接后设置） |
| 设备识别 | 地址中包含厂商 ID `divx1313`（Thorlabs） |
| 实现文件 | `PM100D.py` |

### 2.2 设备地址格式

```
USB0::0x1313::0x8078::P0051073::INSTR
```

或等价格式：

```
USB0::4883::32888::P0047129::0::INSTR
```

| 字段 | 说明 |
|------|------|
| `0x1313` / `4883` | Thorlabs 厂商 ID |
| `0x8078` / `32888` | PM100D 产品 ID |
| `P0051073` | 设备序列号 |

通过 `ResourceManager.list_resources()` 扫描可用地址。

### 2.3 连接初始化

连接成功后软件自动执行：

| 顺序 | 操作 | SCPI 命令 |
|------|------|-----------|
| 1 | 设置功率单位为瓦 | `SENS:POW:UNIT W` |
| 2 | 设置波长为 1550 nm | `SENS:CORR:WAV 1550` |
| 3 | 开启自动量程 | `SENS:CURR:RANG:AUTO 1` |

断开连接时执行：

| 操作 | SCPI 命令 |
|------|-----------|
| 硬件复位 | `*RST` |
| 清除状态 | `*CLS` |

---

### 2.4 SCPI 命令一览

#### 2.4.1 通用命令

| 命令 | 类型 | 说明 | 软件中的用途 |
|------|------|------|--------------|
| `*IDN?` | 查询 | 查询设备标识 | `check_device_status()` |
| `*RST` | 写入 | 硬件复位 | `disconnect()` |
| `*CLS` | 写入 | 清除状态寄存器 | `disconnect()` |

---

#### 2.4.2 功率测量

| 命令 | 类型 | 说明 | 返回值 |
|------|------|------|--------|
| `:MEAS:POW?` | 查询 | 读取当前功率测量值 | 浮点数（单位由 `SENS:POW:UNIT` 决定） |
| `:READ?` | 查询 | 直接读取（代码中注释的备选方案，未启用） | — |

> 当返回值 ≥ `9.9E37` 时，软件视为设备出错，返回 `None`。

采集频率：后台线程约每 **7 ms** 轮询一次 `:MEAS:POW?`。

---

#### 2.4.3 功率单位

| 命令 | 类型 | 说明 | 参数/返回值 |
|------|------|------|-------------|
| `SENS:POW:UNIT <unit>` | 写入 | 设置功率单位 | `W` 或 `DBM` |
| `SENS:POW:UNIT?` | 查询 | 查询当前功率单位 | `W` / `DBM` |

---

#### 2.4.4 波长校正

| 命令 | 类型 | 说明 | 参数/返回值 |
|------|------|------|-------------|
| `SENS:CORR:WAV <wavelength>` | 写入 | 设置工作波长（nm） | 范围：800–1700 |
| `SENS:CORR:WAV?` | 查询 | 查询当前波长（nm） | 浮点数 |

---

#### 2.4.5 输入损耗补偿

| 命令 | 类型 | 说明 | 参数/返回值 |
|------|------|------|-------------|
| `SENS:CORR:LOSS:INP:MAGN <comp>` | 写入 | 设置补偿值（dB） | 范围：-60 ~ +60 |
| `SENS:CORR:LOSS:INP:MAGN?` | 查询 | 查询当前补偿值（dB） | 浮点数 |

---

#### 2.4.6 量程控制

| 命令 | 类型 | 说明 | 参数/返回值 |
|------|------|------|-------------|
| `SENS:CURR:RANG:AUTO 1` | 写入 | 开启自动量程 | — |
| `SENS:CURR:RANG:AUTO 0` | 写入 | 关闭自动量程 | — |
| `SENS:CURR:RANG:AUTO?` | 查询 | 查询自动量程状态 | `1`=开启，`0`=关闭 |
| `SENS:CURR:RANG:UPP <value>` | 写入 | 设置测量上限 | 需在 MIN~MAX 范围内 |
| `SENS:CURR:RANG:UPP?` | 查询 | 查询当前量程上限 | 浮点数 |
| `SENS:CURR:RANG:UPP? MIN` | 查询 | 查询量程下限 | 浮点数 |
| `SENS:CURR:RANG:UPP? MAX` | 查询 | 查询量程上限 | 浮点数 |

---

#### 2.4.7 调零校准

| 命令 | 类型 | 说明 | 参数/返回值 |
|------|------|------|-------------|
| `SENS:CORR:COLL:ZERO:INIT` | 写入 | 启动调零 | — |
| `SENS:CORR:COLL:ZERO:STAT?` | 查询 | 查询调零状态 | `0`=进行中，`1`=完成 |

调零流程：发送 `INIT` 后轮询 `STAT?`，默认超时 3 秒，间隔 0.1 秒。

> 注：调零功能已在驱动中实现（`zero_adjustment()`），当前 GUI 未绑定按钮。

### 2.5 SCPI 与 GUI / TCP 的对应关系

| 软件功能 | SCPI 命令 | TCP 接口 |
|----------|-----------|----------|
| 实时读功率 | `:MEAS:POW?` | `GetPower` |
| 设置波长 | `SENS:CORR:WAV` | 仅 GUI |
| 设置功率单位 | `SENS:POW:UNIT` | 仅 GUI |
| 设置补偿值 | `SENS:CORR:LOSS:INP:MAGN` | 仅 GUI（界面有补偿显示，设置按钮未完全启用） |
| 自动量程开关 | `SENS:CURR:RANG:AUTO` | 仅 GUI |
| 连接设备 | VISA `open_resource` | `ConnectDevice` |
| 数据记录 | —（本地 CSV 写入） | `RecordCon` |
| 版本查询 | — | `check` |

## 三、数据记录文件格式

通过 GUI 或 `RecordCon` 触发的记录，保存为 CSV 文件：

| 项目 | 说明 |
|------|------|
| 目录 | `./Record/` |
| 文件名 | `PowerRecord_{开始时间}.csv`，停止后重命名为 `PowerRecord_{开始时间}_{结束时间}.csv` |
| 编码 | GBK |
| 列 | `时间,功率,波长,补偿值` |

时间格式：`YYYY-MM-DD HH:MM:SS.mmm`（毫秒三位）。

## 四、注意事项

1. **TCP 端口占用**：端口 `1235` 被占用时服务器无法启动，需关闭占用进程或修改源码中的 `auto_port`。
2. **设备资源锁**：设置波长、单位、量程等操作期间，采集线程会暂停（`operation` 标志），避免 SCPI 指令冲突。
3. **断开重连**：断开时先停止采集线程（最多等待 3 秒），再发送 `*RST`/`*CLS`，避免 `VI_ERROR_RSRC_LOCKED`。
4. **GetPower 返回值**：返回的是界面 LCD 格式化后的字符串，非原始 SCPI 浮点值；如需原始值请直接通过 VISA 读取 `:MEAS:POW?`。
5. **ConnectDevice 限制**：远程连接命令使用 GUI 当前选中的设备，不支持在 TCP 请求中指定设备地址。
